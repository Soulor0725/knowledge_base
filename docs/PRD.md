# 知识库管理系统 - 产品需求文档（PRD）

## 一、产品概述

### 1.1 产品名称
智慧知识库管理系统（Knowledge Hub）

### 1.2 产品定位
面向个人和团队的知识管理工具，帮助用户高效收集、整理、检索和分享知识内容。

### 1.3 目标用户
- 个人知识管理者
- 技术团队
- 产品经理
- 运营人员
- 学生/研究人员

### 1.4 核心价值
- **高效整理**：多维度分类、标签管理
- **快速检索**：全文搜索、智能推荐
- **安全存储**：云端备份、权限控制
- **便捷分享**：团队协作、知识共享

### 2.2 功能详细说明

#### 用户中心
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 用户注册 | 支持用户名+密码注册 | P0 |
| 用户登录 | JWT Token认证 | P0 |
| 个人信息编辑 | 修改昵称、头像 | P1 |
| 密码修改 | 安全修改密码 | P2 |

#### 知识管理
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 创建文章 | 支持Markdown编辑 | P0 |
| 编辑文章 | 实时保存草稿 | P0 |
| 删除文章 | 软删除/硬删除 | P0 |
| 文章分类 | 多级分类管理 | P0 |
| 文章标签 | 多标签支持 | P1 |
| 全文搜索 | 标题+内容搜索 | P0 |
| 收藏功能 | 一键收藏/取消 | P1 |
| 文章置顶 | 重要文章置顶 | P2 |
| 浏览计数 | 自动统计浏览量 | P1 |

#### 数据分析
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 总览统计 | 文章数、浏览量、收藏数 | P1 |
| 分类统计 | 各分类文章占比 | P2 |
| 趋势分析 | 创建时间分布 | P2 |
| 导出报表 | CSV/Excel导出 | P3 |

#### 猕猴桃销售管理
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 订单列表 | 分页展示销售记录 | P0 |
| 新增订单 | 客户信息、规格、数量、金额 | P0 |
| 编辑订单 | 修改订单信息 | P0 |
| 删除订单 | 删除销售记录 | P0 |
| 状态管理 | 已发货/未发货状态切换 | P0 |
| 运单号管理 | 填写物流单号 | P1 |
| 订单搜索 | 按客户名、手机号搜索 | P1 |
| 销售统计 | 总订单数、总销量、总金额 | P1 |
| 报表导出 | 按客户汇总统计 | P2 |

**订单字段说明：**
- 客户名（必填，最长50字符）
- 手机号（必填，11位数字）
- 收货地址（必填，最长200字符）
- 接单日期（必填）
- 规格（5斤装/10斤装）
- 数量（正整数）
- 支付金额（非负数，保留2位小数）
- 状态（已发货/未发货）
- 运单号（可选，最长50字符）

#### 加班记录管理
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 记录列表 | 分页展示加班记录 | P0 |
| 新增记录 | 日期、时间、类型、备注 | P0 |
| 编辑记录 | 修改加班信息 | P0 |
| 删除记录 | 单条/批量删除 | P0 |
| 时长计算 | 自动计算加班时长 | P0 |
| 月度统计 | 工作日/周末加班统计 | P1 |
| 月份筛选 | 按月份筛选记录 | P1 |
| 类型筛选 | 工作日/周末类型筛选 | P2 |

**加班规则：**
- 工作日加班：19:00后开始计算，结束时间19:00-23:59
- 周末加班：09:00-23:00，自动扣除12:00-14:00午餐时间（2小时）
- 同一天只能有一条加班记录

## 四、交互设计规范

### 4.1 交互模式

| 场景 | 交互方式 |
|------|----------|
| 创建文章 | 表单提交 + 实时草稿保存 |
| 删除文章 | 二次确认弹窗 |
| 搜索 | 实时搜索 + 防抖 |

## 五、数据架构

### 5.1 数据库设计

#### 用户表（users）
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL,
    password TEXT NOT NULL,
    name TEXT,
    avatar TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 文章表（articles）
```sql
CREATE TABLE articles (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    content TEXT NOT NULL,
    category TEXT DEFAULT '未分类',
    tags TEXT DEFAULT '',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    views INTEGER DEFAULT 0,
    is_favorite INTEGER DEFAULT 0,
    is_draft INTEGER DEFAULT 0,
    user_id INTEGER,
    FOREIGN KEY (user_id) REFERENCES users (id)
);
```

#### 分类表（categories）
```sql
CREATE TABLE categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT UNIQUE NOT NULL,
    color TEXT DEFAULT '#667eea',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 猕猴桃销售表（kiwi_sales）
```sql
CREATE TABLE kiwi_sales (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    customer_name TEXT NOT NULL,
    phone TEXT NOT NULL,
    address TEXT NOT NULL,
    order_date TEXT NOT NULL,
    status TEXT DEFAULT '未发货',
    tracking_number TEXT,
    remark TEXT,
    quantity INTEGER DEFAULT 0,
    payment_amount REAL DEFAULT 0.00,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    user_id INTEGER,
    FOREIGN KEY (user_id) REFERENCES users (id)
);
```

#### 加班记录表（overtime_records）
```sql
CREATE TABLE overtime_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    overtime_type TEXT NOT NULL,
    date TEXT NOT NULL,
    start_time TEXT NOT NULL,
    end_time TEXT NOT NULL,
    duration REAL NOT NULL,
    remark TEXT DEFAULT '',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    user_id INTEGER,
    FOREIGN KEY (user_id) REFERENCES users (id)
);
```
