### 1.2 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | HTML/CSS/JavaScript | 原型阶段使用静态页面 |
| 后端 | Python Flask 3.0+ | 轻量级Web框架 |
| 数据库 | SQLite | 轻量级嵌入式数据库 |
| 认证 | JWT | 无状态Token认证 |
| 密码加密 | PBKDF2-SHA256 | 安全的密码哈希 |

```

│  │ 新建    │───▶│ 未发货  │───▶│ 已发货  │                 │
│  │ 订单    │    │         │    │         │                 │
│  └─────────┘    └─────────┘    └─────────┘                 │
│                       │              │                      │
│                       │              ▼                      │

│                       │         │ 已完成  │                 │

**API接口：**
- `GET /api/kiwi-sales` - 获取销售列表（支持分页、搜索）
- `POST /api/kiwi-sales` - 新增销售记录
- `PUT /api/kiwi-sales/:id` - 更新销售记录
- `DELETE /api/kiwi-sales/:id` - 删除销售记录
- `GET /api/kiwi-sales-report` - 获取销售报表（按客户汇总）

**业务规则：**
- 客户名：必填，最长50字符
- 手机号：必填，11位数字
- 地址：必填，最长200字符
- 接单日期：必填
- 规格：5斤装/10斤装
- 数量：正整数
- 金额：非负数，保留2位小数
- 状态：已发货/未发货

│  ┌─────────────────────────────────────────────────────┐   │
│  │  19:00 ──────────────────────────────────── 结束时间 │   │
│  │  (固定起点)          计算时长                        │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  周末加班：                                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  开始时间 ──── 12:00-14:00 ──── 结束时间            │   │
│  │               (午餐扣除)                            │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**API接口：**
- `GET /api/overtime` - 获取加班列表（支持分页、月份筛选）
- `POST /api/overtime` - 新增加班记录
- `PUT /api/overtime/:id` - 更新加班记录
- `DELETE /api/overtime/:id` - 删除加班记录
- `POST /api/overtime/batch-delete` - 批量删除加班记录
- `GET /api/overtime/stats` - 获取月度加班统计

**业务规则：**
- 工作日加班：结束时间必须在19:00-23:59范围
- 周末加班：时间范围09:00-23:00，自动扣除12:00-14:00午餐时间
- 同一天只能有一条加班记录
- 时长自动计算，保留1位小数

## 四、安全设计

### 4.1 认证授权流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                      JWT认证流程                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  客户端                        服务端                               │
│  ┌─────┐                       ┌─────┐                             │
│  │     │──── 登录请求 ────────▶│     │                             │
│  │     │                       │     │                             │
│  │     │                       │     │── 验证用户名密码             │
│  │     │                       │     │                             │
│  │     │◀─── 返回Token ────────│     │                             │
│  │     │                       └─────┘                             │
│  │     │                                                          │
│  │     │──── 携带Token请求 ──▶┌─────┐                             │
│  │     │                      │     │                             │
│  │     │                      │     │── 验证Token                 │
│  │     │                      │     │                             │
│  │     │◀─── 返回数据 ────────│     │                             │
│  │     │                      └─────┘                             │
│  └─────┘                                                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.2 安全措施

| 安全层面 | 措施 | 说明 |
|----------|------|------|
| 认证 | JWT Token | 7天有效期，无状态认证 |
| 密码 | PBKDF2-SHA256 | 防暴力破解 |
| 授权 | 用户隔离 | 只能访问自己的数据 |
| 输入 | 参数验证 | 防止SQL注入和XSS |
| 传输 | HTTPS | 加密传输（生产环境） |
